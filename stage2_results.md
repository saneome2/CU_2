# Результаты выполнения этапа 2: Сбор данных

## Описание
Реализована логика получения прямых зависимостей пакета из APK репозитория Alpine Linux. Информация извлекается из APKINDEX.tar.gz файла, парсится вручную без использования сторонних библиотек.

## Реализованная функциональность
- Скачивание и распаковка APKINDEX.tar.gz из указанного репозитория.
- Парсинг метаданных пакетов для поиска заданного пакета и версии.
- Извлечение списка прямых зависимостей.

## Демонстрация
Пример запуска для пакета `busybox` версии `1.36.1-r7` из репозитория `https://dl-cdn.alpinelinux.org/alpine/v3.18/main/`:

```
python main.py --package-name busybox --repo-url https://dl-cdn.alpinelinux.org/alpine/v3.18/main/ --version 1.36.1-r7
```

Вывод:
```
Прямые зависимости:
so:libc.musl-x86_64.so.1
```

Если пакет или версия не найдены, выводится "Зависимости не найдены."

## Обработка ошибок
- Ошибки при скачивании или обработке APKINDEX выводятся с сообщением.
- Если версия не указана, выводится ошибка "Версия пакета обязательна."
