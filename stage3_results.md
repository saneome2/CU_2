# Результаты выполнения этапа 3: Основные операции

## Описание
Реализован алгоритм построения графа зависимостей с использованием итеративного DFS (без рекурсии). Корректно обрабатываются циклические зависимости. Добавлена поддержка тестового режима с файлом описания графа (пакеты - большие буквы латинского алфавита).

## Реализованная функциональность
- Итеративный DFS для сбора всех транзитивных зависимостей.
- Обработка циклов: предотвращение бесконечной рекурсии в выводе дерева.
- Тестовый режим: чтение графа из файла (формат "A: B C").
- Вывод ASCII-дерева зависимостей (если --ascii-tree).
- Сохранение списка зависимостей в файл (иначе).

## Демонстрация на тестовых случаях

### Случай 1: Простой граф без циклов (пакет A)
Файл `test_graph.txt`:
```
A: B C
B: D
C: D
D:
```

Команда:
```
python main.py --package-name A --repo-url test_graph.txt --test-mode --version dummy --ascii-tree
```

Вывод:
```
└── A
    ├── B
    │   └── D
    └── C
        └── D
```

### Случай 2: Граф с циклом (пакет F)
Файл `test_graph.txt` (добавлены F: G, G: F).

Команда:
```
python main.py --package-name F --repo-url test_graph.txt --test-mode --version dummy --ascii-tree
```

Вывод:
```
└── F
    └── G
        └── F (cycle)
```

### Случай 3: Сохранение в файл (пакет A)
Команда:
```
python main.py --package-name A --repo-url test_graph.txt --test-mode --version dummy
```

Вывод:
```
Граф зависимостей сохранён в graph.png
```

Содержимое `graph.png`:
```
Dependency graph for A:
B
C
D
```

## Обработка ошибок
- Циклы предотвращены с помощью множества visited.
- Ошибки чтения файла выводятся с сообщением.
